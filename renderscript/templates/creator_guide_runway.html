What you do now

  1. Prepare your reference images
  2. Generate takes for each shot
  3. Mark keepers
  4. Edit picture first
  5. Add audio in post

Golden rule

Picture-first.

No burned-in subtitles, captions, logos, watermarks, or on-screen text.

If text appears in a generated shot, discard that take and generate again.

Audio comes later: ADR / VO, sound effects, and mix are added after the rough cut.

Generation workflow

Open your AI video tool → apply references → paste the shot prompt → generate → mark keeper

Choose your starting workflow

Path A — Prompts-only (quick test) Use this when you don’t have reference images yet. Generate a few shots to test pacing and visual intent. Expect more drift at this stage — this is normal.

Path B — References (best consistency) Use this when you can provide reference images. References improve consistency more than complex prompts. This package shows which references each shot expects.

Workflow map

Scene ↓ Generate refs ↓ Generate shot_001, shot_002, ... ↓ Mark keepers ↓ Edit ↓ Audio in post

Takes: Generate shots

This is your dailies stage: generate multiple takes per shot, then pick a keeper.

Non‑negotiable

No burned-in subtitles or on-screen text. Keep picture clean.

If your tool adds text, reroll the take.

Shot loop (repeat for each shot_###)

1) Open the shot prompt block

2) Apply refs (if using Path B)

3) Generate 3–6 takes

4) Review fast

5) Mark a keeper

If drift rises: re-apply refs → simplify prompt → split the shot.

Page 5

Edit → Audio (post)

Cut the scene first. Then build audio like a normal production.

Rough cut (picture)

Assemble keepers, focus on timing and story. Don’t polish audio yet—get the scene working visually.

Audio in post

• Dialogue (ADR/VO): use the screenplay source and the rendered package guide.

• SFX/ambience: track notes in your editing project or external sound workflow.

• Mix: dialogue first, SFX second, music last

• Keep audio decisions outside the standard RenderPackage unless your workflow adds them separately.
