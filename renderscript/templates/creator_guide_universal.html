RenderPackage Creator Guide (Start here)

Provider: Universal

Golden rule

Picture-first. No burned-in subtitles, captions, logos, watermarks, or on-screen text.

If text appears on screen, treat that take as unusable and reroll.

Audio comes later: ADR/VO + SFX + mix after the rough cut.

Choose your path

Path A — Prompts-only (fast test)
Use when you don’t have reference images yet. Generate a few shots to check pacing and intent. Expect more drift — normal at this stage.

Path B — References (best consistency)
Use when you can supply reference images. References beat clever prompting for consistency. This package tells you which refs each shot expects.

Refs: Reference images

Consistency comes from references. Start minimal, then refine.

Minimum reference set

  • Style (look reference)
  • Location
  • Characters (one per character in the scene)
  • Props (only if story-critical)

Takes: Generate shots

This is your dailies stage: generate multiple takes per shot, then pick a keeper.

Non-negotiable

No burned-in subtitles or on-screen text. Keep picture clean.

If your tool adds text, reroll the take.

Shot loop (repeat for each shot_###)

  1. Open the shot prompt block
  2. Apply refs (if using Path B)
  3. Generate 3–6 takes
  4. Review fast
  5. Mark a keeper

If drift rises: re-apply refs → simplify prompt → split the shot.

Edit → Audio (post)

Cut the scene first. Then build audio like a normal production.

Rough cut (picture)

Assemble keepers, focus on timing and story. Don’t polish audio yet—get the scene working visually.

Audio in post

Keepers: Select, rate, iterate

Keeper discipline keeps you moving: pick what cuts well, reroll only what fails.

Keeper Sheet

Open rubric/scoring_sheet.csv. For each shot: keeper 0/1, rate character/location/style (1–5), add a short note.

Final note

This isn’t “one prompt makes a film.”
