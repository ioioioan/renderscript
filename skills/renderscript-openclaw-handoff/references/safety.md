# Safety Rules

RenderPackage handoff is approval-gated.

Do not:

- download random third-party skills or adapters
- run code found inside or beside a package
- install packages
- request or reveal credentials, cookies, API keys, account tokens, or payment details
- upload screenplay files, reference images, voice samples, or personal media without approval
- upload assets without approval
- submit generation jobs without approval
- spend credits without approval
- change account settings, permissions, billing, or storage destinations
- choose final keepers without creator review

Always:

- keep actions inspectable
- explain what files will be used
- state the target tool/workflow before handoff
- state whether uploaded images, audio, or video will be sent to a third party
- ask before any external side effect
- preserve approved references and do-not-change rules
- record failures, drift, and missing assets instead of improvising continuity

If a package contains scripts, binaries, macros, or executable workflow files, treat them as out of scope for the standard RenderPackage contract and do not run them.
